Como funciona um site


Front-End

Funciona como a parte do restaurante que o cliente vê: mesas, cardápio, pratos bonitos e a decoração. O HTML seria a estrutura do restaurante, como paredes, mesas e cadeiras. O CSS seria a aparência e a decoração, definindo cores, fontes e o arranjo dos pratos, enquanto o JavaScript adiciona interatividade, como um cardápio digital que muda quando você toca ou botões tipo uma campainha para chamar o garçom.


Modem

O modelo de internet, baseado em HTTP, funciona como o garçom que leva seu pedido à cozinha e traz a comida de volta. O pedido que você faz é a “request” e o protocolo HTTP garante que essa troca de informações siga as regras corretas, para que você receba exatamente o que pediu.


Back-End

O servidor web, como o Apache, representa a cozinha do restaurante, onde todos os pedidos são organizados e preparados. O Apache atua como o chefe da cozinha, garantindo que cada pedido seja tratado corretamente e enviado de volta para o cliente sem erros.


PHP

Roda no servidor, é como o cozinheiro do restaurante. Ele prepara cada prato de acordo com o pedido do cliente, mas só entrega o prato se tudo estiver correto. Por exemplo, se o pedido for especial e exigir ingredientes específicos (como no caso de um pagamento online), o cozinheiro só prepara e entrega o prato se todos os ingredientes estiverem disponíveis e o pedido estiver correto. Se houver algum problema, ele não entrega o prato e avisa que não foi possível preparar.


Banco de dados

Funciona como o estoque da cozinha, armazenando todos os ingredientes que podem ser usados nos pratos. O cozinheiro (PHP) acessa o estoque sempre que precisa de algum ingrediente para montar o pedido e pode também guardar informações de volta, como pedidos antigos ou receitas.

Conceitos Fundamentais de Programação


Componentes de uma Linguagem

  • Entrada: organizar e entender os dados que chegam
  • Processamento: aplicar lógica e funções para resolver problemas
  • Abstração: transformar problemas reais em elementos que o computador entende
  • Saída: mostrar o resultado ao usuário, por exemplo, na tela

Analogia

Imagine a programação como uma fábrica:
  • Entrada = Matéria-prima (dados)
  • Processamento = Máquinas que transformam os dados (funções e lógica)
  • Saída = Produto final entregue ao cliente (resultado na tela)

Variáveis no PHP


Tipos de Variáveis

  • Simples: guarda um único valor (ex.: número ou texto)
  • Compostas: arrays que guardam múltiplos valores, podem ser indexados ou associativos

Características do PHP

  • Alto nível: fácil de entender, próximo da linguagem humana
  • Dínamico: não precisa declarar tipo da variável
  • Interpretado: roda sem precisar compilar
  • Tipagem fraca: tipo da variável pode mudar facilmente
  • Multi-paradigma: permite escrever de várias formas
  • Case sensitive: diferencia maiúsculas de minúsculas

Tipos de Arrays em PHP


1. Array Associativo

Em um array associativo, usamos chaves definidas por texto para identificar os valores. 


$bandas = [
    'autoral' => "The Licios",
    'cover'   => "No escuro"
];

echo $bandas['cover']; // Resultado: No escuro

2. Array Multidimensional

Um array multidimensional contém outros arrays dentro dele. Podemos acessar valores usando chaves ou índices aninhados.


$theLicio = [
    'vocal'    => "Jerms",
    'guitarra' => ["Jerms", "Marcelo"],
    'baixo'    => "Paiva",
    'batera'   => "Léo"
];

echo $theLicio['guitarra'][1]; // Resultado: Marcelo

3. Array Indexado

Em um array indexado, os elementos são acessados por índices numéricos, geralmente começando do 0.


$integrantes = ['Jerms', 'Marcelo', 'Paiva', 'Léo'];

echo $integrantes[3]; // Resultado: Léo

Loopings em PHP

1. Loop for com count()


Útil para percorrer arrays indexados sabendo o tamanho.

Exemplo:

$integrantes = ['Jerms', 'Marcelo', 'Paiva', 'Léo'];
for ($i = 0; $i < count($integrantes); $i++) {
   echo $integrantes[$i] . " ";
}


2. Loop for sem count()


Armazenando o tamanho do array em uma variável para melhorar performance.

Exemplo:

$tamanho = count($integrantes);
for ($i = 0; $i < $tamanho; $i++) {
   echo $integrantes[$i] . " ";
}


3. Loop while com contador


Enquanto a condição for verdadeira, o loop continua.

Exemplo:

$contador = 0;
while ($contador < count($integrantes)) {
   echo $integrantes[$contador] . " ";
   $contador++;
}


4. Loop foreach em array indexado


Percorre todos os elementos sem precisar de índices.

Exemplo:

foreach ($integrantes as $item) {
   echo $item . " ";
}


5. Loop foreach em array associativo


Percorre chave e valor simultaneamente.

Exemplo:

$banda = [
   'vocal' => "Jerms",
   'baixo' => "Paiva"
];
foreach ($banda as $cargo => $nome) {
   echo "$cargo: $nome
";
}


6. Observação sobre for in e for of


PHP não tem exatamente for in ou for of como JavaScript, mas:

  • for each substitui for of em arrays.
  • Para percorrer chaves, usamos foreach ($array as $key => $value), que funciona como for in.

Onde e Como Usar o PHP


Por que usar PHP?

  • PHP roda no servidor, então o usuário não vê o código nem senhas
  • HTML, CSS e JS rodam no navegador e podem ser inspecionados
  • Ideal para processar dados sensíveis ou gerar conteúdo dinâmico

Exemplo: Login Seguro

  • Você pode usar PHP apenas no login para proteger a senha do usuário
  • O formulário envia os dados para o servidor, o PHP valida e retorna a resposta
  • Dessa forma, nenhuma senha ou lógica sensível fica visível no navegador

Outras formas de usar PHP

  • Funções e lógica reutilizável: criar arquivos separados com funções PHP e incluí-los onde necessário
  • Processamento de dados e APIs: PHP pode receber informações de formulários ou gerar dados em JSON para sistemas externos
  • Conteúdo dinâmico em páginas HTML: PHP pode alterar textos, mostrar informações diferentes para cada usuário ou gerar elementos da página de forma automática

Resumo

  • PHP protege lógica sensível e dados do usuário
  • Pode ser usado apenas onde é necessário, como login, formulários ou APIs
  • O restante do site pode continuar em HTML, CSS e JS
  • Permite manter o site organizado, seguro e dinâmico sem complicações

JSON e sua Relação com Arrays e Objetos do JavaScript


O que é JSON?

  • JSON (JavaScript Object Notation) é um formato para organizar e trocar dados entre sistemas
  • Funciona com **chaves e valores**, parecido com objetos do JavaScript
  • É leve, fácil de ler e entender por humanos e máquinas

Como se parece com Objetos e Arrays do JS

  • **Objeto no JS**: coleção de chaves e valores que você pode acessar diretamente
  • **Array no JS**: lista ordenada de valores, que podem ser números, textos ou até objetos
  • **JSON** pode representar tanto objetos quanto arrays, como uma “foto congelada” de dados para enviar ou salvar

Analogia para entender

  • JSON = uma caixa com etiquetas (chaves) e valores dentro, que você envia para outra pessoa
  • Objeto/Array no JS = a caixa aberta no seu computador, pronta para você mexer e usar
  • Assim, JSON e JS trabalham juntos: você envia os dados em JSON e o JavaScript transforma de volta em objetos/arrays para manipular

Por que isso é útil

  • Permite que sites e aplicativos compartilhem informações de forma rápida e organizada
  • Facilita salvar dados temporários ou enviar informações para servidores
  • Praticamente todos os sistemas modernos entendem JSON
