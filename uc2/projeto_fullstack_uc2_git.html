Git e GitHub

Usar ferramentas como essas no desenvolvimento de um projeto, garante controle, segurança e organização do código, permitindo registrar cada etapa do projeto, trabalhando em equipe sem gerar conflitos e mantendo cópias sincronizadas salvas na nuvem. Com isso, é possível evoluir o código com facilidade e colaboração de forma mais profissional.

Ferramentas

  • Git: Software de controle de versão de código, instalado no computador e que funciona localmente, sem necessidade de conexão com a internet. Pode ser integrado a editores como o VS Code, facilitando o acompanhamento e registro das alterações diretamente no software de desenvolvimento.

  • GitHub: Plataforma online para armazenar e colaborar em projetos com Git.

  • GitHub Desktop: Programa instalado no computador que facilita o uso do Git e integração com o GitHub.

Repositório

Repositório é uma pasta com controle de versão ativado pelo Git, usada para armazenar o código-fonte e acompanhar todas as mudanças feitas no projeto ao longo do tempo. Pode ser local (no seu computador) ou remoto (hospedado em plataformas como o GitHub).

Registros

  • Commit: Registro de uma versão (ponto no tempo) do código.
  • Push: Enviar as alterações para o GitHub (nuvem).
  • Pull: Receber alterações do GitHub para sua máquina.

Status do arquivo

  • Novo: ainda não versionado.
  • Modificado: foi alterado.
  • Salvo (commitado): já está registrado.

Ramificação

  • Main ou Master: Branch principal.
  • Branch: Linha paralela de desenvolvimento.
  • Merge: Unir alterações de diferentes branches.

Arquivos especiais

  • README.md: Arquivo de apresentação e informações do projeto.
  • Markdown (.md): Linguagem de marcação para escrever o README e outros docs.
  • .gitignore: Arquivo que define o que não será enviado ao GitHub (ex: arquivos grandes, senhas, etc).

Boas Práticas

  • Commits pequenos e frequentes.
  • Mensagens claras e específicas.
  • Revisar antes de enviar (push).
  • Evitar enviar arquivos desnecessários, grandes ou confidenciais.

Outros Conceitos

  • Update: Atualizar repositório local com as mudanças remotas.
  • Clone: Copiar um repositório do GitHub para o seu computador.
  • Novo Repositório: Nome sem espaços e tudo minúsculo, e adicionar descrição clara para facilitar a busca online.
