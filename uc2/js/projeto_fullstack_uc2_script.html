Programar JavaScript no Final do Programa


  • Sequência recomendada: HTML, CSS e JavaScript por último.

Explicação:
Colocar o JavaScript no final do HTML garante que todos os elementos da página sejam carregados antes da execução dos scripts,
evitando erros de manipulação de elementos que ainda não existem.


Resumo:
Sempre colocar o JS após HTML e CSS ou usar "defer" se for no .

Preferência de Operações Aritméticas


  • Expressões complexas devem usar parênteses para priorizar operações.

Explicação:
Parênteses definem a ordem de execução,
evitando erros em cálculos matemáticos ou concatenações que envolvam múltiplas operações.


Resumo:
Sempre use parênteses para garantir que a expressão seja calculada corretamente.

Formas de Concatenar Strings


Forma 1: Concatenação Tradicional

  • Usa aspas duplas e operador +
let nome = "Tiago";
let curso = "JS";
console.log("Olá " + nome + ", " + curso + "!");
// Saída: Olá Tiago, JS!

Explicação:
Une strings e variáveis com +.
Ideal para textos curtos, evitando que o código estoure a largura da tela.


Forma 2: Template Literal / String

  • Usa crase `` e ${variável}
let nome = "Tiago";
let curso = "JS";
console.log(`Olá ${nome}, ${curso}!`);
// Saída: Olá Tiago, JS!

Explicação:
Template literals inserem variáveis diretamente na string usando ${}.
Mais legível que o +, mesmo para pequenas frases.


Concatenação com +=

let msg = "Oi ";
msg += "JS!";
console.log(msg);
// Saída: Oi JS!

Explicação:
+= acrescenta conteúdo a uma string já existente de forma simples.


Resumo:
Strings podem ser unidas com +, += ou template literals.
Para frases curtas e telas estreitas, use exemplos minimalistas.

Dados Estáticos e Dinâmicos


  • Estático = dado fixo → declarar com const
  • Dinâmico = dado alterável → declarar com let

Explicação:
Constantes não podem ser reatribuídas,
garantindo integridade de dados fixos, enquanto variáveis let podem ser atualizadas ao longo do programa.


Resumo:
Use const para valores fixos
e let para valores que mudam.

Templates Literais e typeof


  • Templates literais usam crase `` e ${variável} para interpolar valores.
  • typeof retorna o tipo de uma variável.

Explicação:
Template literals simplificam a concatenação e formatação de strings.
typeof ajuda a identificar se uma variável é string, number, boolean, etc.


Resumo:
Template literals e typeof são ferramentas essenciais
para manipulação e inspeção de variáveis.

JavaScript Externo e Ordem de Carregamento


  • Linkar JS externo no final do body usando <script src="arquivo.js"></script>.
  • Se for no head, usar defer para carregar a página primeiro.
  • Sempre garantir que o script rode depois que o HTML foi carregado.

Explicação:
Executar scripts antes de o HTML estar pronto pode causar erros de manipulação de elementos.
defer resolve isso quando o script está no .


Resumo:
Colocar o JS no final do body ou usar defer é a prática recomendada.

Uso Correto de Aspas em Strings


// ERRADO
        // console.log("Gosto "muito" de paçoca");

        // CORRETO
        console.log('Gosto "muito" de paçoca');
        console.log("Gosto 'muito' de paçoca");
        console.log("Gosto \"muito\" de paçoca");

Explicação:
Use aspas simples dentro de aspas duplas ou vice-versa,
ou escape com barra invertida (\) para evitar erros de sintaxe.


Resumo:
Sempre harmonize o uso de aspas para strings
e escape quando necessário.

Modo Estrito em JavaScript


// Habilitando o modo estrito
        'use strict';

        teste = "Senac";
        console.log(teste);

Explicação:
O modo estrito força boas práticas,
prevenindo o uso de variáveis não declaradas e o uso incorreto de palavras reservadas.
Ajuda a detectar erros mais cedo.


Resumo:
Sempre inicie arquivos JS com 'use strict'
para código mais seguro e confiável.
