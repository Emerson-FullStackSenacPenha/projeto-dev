Conceito e Tipos de Dados

Em programação, dados podem ser organizados de duas formas principais: Relacionais e Não Relacionais.


Relacionais

Organizados em tabelas, linhas e colunas. Cada linha + tabela forma um registro.


  • Tabela: conjunto de registros
  • Linha: cada registro
  • Campo: nome de cada coluna (ex: nome, telefone, e-mail)

Não Relacionais

Organização flexível, como arrays e objetos, podendo ter objetos dentro de objetos ou arrays.


  • Ex: MongoDB
  • Redis

Observação: sessões em PHP e JS armazenam dados temporariamente na máquina do usuário.

Modelos de Banco de Dados


No curso, estudamos principalmente o Modelo Relacional, que organiza os dados em tabelas, linhas e colunas. Cada tabela representa uma entidade, cada linha um registro, e cada coluna um atributo.

Analogia: é como uma planilha do Excel, onde cada linha é um item e cada coluna uma característica.


Outros Modelos Comuns


  • Não Relacional (NoSQL): mais flexível, como objetos ou arrays dentro de objetos. Ex: MongoDB.
  • Hierárquico: dados organizados em árvore, como pastas e subpastas.
  • Em Rede: similar ao hierárquico, mas permite múltiplas conexões entre os registros.

Modelagem de Dados

Modelagem ajuda a organizar dados de forma lógica antes de criar o banco.


Modelagem Conceitual

Usa o MER (Modelo de Entidade e Relacionamento).


  • Entidade: algo real, ex: Cliente
  • Atributos: características da entidade, ex: nome, telefone
  • Relacionamentos: como entidades se conectam, ex: Cliente faz Pedido

DER (Diagrama Entidade-Relacionamento)


  • Retângulo = Entidade
  • Elipse = Atributo
  • Losango = Relacionamento (pode ser uma tabela pivô)
  • Identificador = sublinhado (ex: ID)

Cardinalidade


  • 1:1 = um para um
  • 1:N = um para muitos
  • N:N = muitos para muitos

Entidade associativa ajuda a representar relacionamentos N:N.

Terminologias e Conceitos


  • Consulta/Query: comandos SQL para buscar dados
  • Comando SELECT: busca dados no banco
  • CRUD: Create, Read, Update, Delete – manipulação de dados
  • BackUp e Restauração: segurança dos dados
  • PDV = Ponto de Vendas
  • Diagrama = documentação do banco de dados

Dicas de Ferramentas

Modelagem Lógica de Banco de Dados


Modelagem lógica é como desenhar o mapa da sua cidade, mas no mundo dos dados. Você define onde ficam as “casas” (tabelas) e como as “ruas” (relacionamentos) conectam essas casas.


Entidades

As entidades são as tabelas do banco de dados. Cada tabela representa algo que você quer guardar informações.


Atributos

São as colunas das tabelas. Cada coluna tem um tipo de dado específico, como números ou texto.


Relacionamentos

São as conexões entre tabelas. Usamos chaves primárias e estrangeiras para criar esses vínculos.

SQL e Tipagem


SQL é a linguagem usada para conversar com o banco de dados, pense nela como a língua oficial da cidade dos dados. É importante declarar variáveis e usar tipagem forte, ou seja, cada dado deve ter um tipo específico.


Tipos de Dados


  • Texto: varchar (Tamanho variável), char (tamanho fixo), text
  • Numéricos: int, decimal, float
  • Datas/Períodos: date, datetime, timestamp

Chaves no Banco de Dados


As chaves ajudam a organizar e conectar informações entre tabelas.


Chave Primária (PK)

É o identificador único da tabela. Exemplo: id


Chave Estrangeira (FK)

É a referência para a chave primária de outra tabela, permitindo relacionamentos.

Boas Práticas de Nomenclatura


  • Entidade: nome em minúsculo e no plural
  • Atributo: nome em minúsculo e no singular
  • Tipo de dado: maiúsculo
  • Not Null: define que o campo não pode ser nulo
  • AUTO_INCREMENT: o banco gera automaticamente um identificador
  • DECIMAL(10,2): 10 dígitos no total, sendo 2 dígitos para a parte decimal e 8 dígitos para a parte inteira.

Relacionamentos Avançados


Quando duas tabelas precisam de uma ligação mais complexa, usamos chaves primárias compostas.

Exemplo: uma tabela de vendas que relaciona cliente e produto simultaneamente.

Ferramentas e Ambientes


MySQL é uma IDE (interface) que facilita escrever e executar SQL, que é a linguagem usada para manipular os dados.

Banco de Dados


Um banco de dados é como um **armário organizado** onde guardamos informações. Cada tipo de dado precisa de um "espaço" adequado:


Tipos de dados importantes


  • INT = números inteiros
  • VARCHAR(255) = texto com tamanho variável, como nomes ou e-mails
  • CHAR = texto com tamanho fixo
  • TEXT = textos longos
  • DECIMAL = números com casas decimais, como preços
  • DATE, TIME, DATETIME = datas, horários ou ambos
  • ENUM('admin','editor') = lista de opções pré-definidas
  • BLOB = usado para guardar arquivos, como imagens ou PDFs

Armazenamento Seguro de Senhas em PHP


O que é Hash?

Um hash é como uma “impressão digital” da sua senha, ele transforma qualquer senha em uma sequência fixa de caracteres. Essa transformação é unidirecional, não dá para descobrir a senha original a partir do hash.


Exemplo de Hash

  • Senha: 123456
  • Hash SHA-256: 8d969eef6ecad3c29a3a629280e686cff8fab...

Por que usar Hash?

  • Segurança: protege a senha real mesmo se o banco for acessado por terceiros.
  • Proteção contra vazamentos: difícil descobrir a senha original a partir do hash.

Armazenando Hash no Banco de Dados

Para armazenar hashes no MySQL, use uma coluna VARCHAR(255).
Isso garante que qualquer hash moderno, como os gerados por password_hash() no PHP, caiba na coluna sem problemas.


Resumo Didático

Resumindo: nunca guarde senhas em texto puro.
Sempre transforme a senha em hash e armazene em VARCHAR(255) no banco.
Assim, mesmo em caso de ataque, suas senhas ficam protegidas.

Normalização de Banco de Dados


É como **arrumar uma prateleira de livros**: organizamos para não ter livros duplicados e facilitar futuras alterações. Benefícios:
  • Evita registros duplicados
  • Evita inconsistência de informações
  • Facilita futuras atualizações e manutenção

Modelagem Física em SQL

É o mapa real do banco de dados, onde transformamos a ideia (modelagem lógica) em algo que o computador entende.


SGBD - Sistema Gerenciador de Banco de Dados

Programas que armazenam e organizam os dados, como MySQL, SQLite, PostgreSQL, SQL Server e Oracle. Eles usam SQL (Linguagem de Consulta Estruturada) para trabalhar com os dados.


Principais comandos SQL

  • DDL (Definição) = criar e alterar estruturas: CREATE, ALTER
  • DML (Manipulação) = trabalhar com dados: INSERT, UPDATE, DELETE
  • DQL (Consulta) = buscar dados: SELECT

Estrutura básica da modelagem física


Criar banco de dados

CREATE DATABASE nome_do_banco CHARACTER SET utf8mb4;

Selecionar banco para uso

USE nome_do_banco;

Criar tabela


Estrutura de tabela define **colunas, tipos de dados e restrições**:
  • Primary Key = identificador único
  • Foreign Key = referência a outra tabela
  • NOT NULL = campo obrigatório

Cuidados
